foreman_host¶

Description¶

A host managed by Foreman.

Interactivity¶

  • Creatable: YES
  • Deletable: YES
  • Mutable: YES

Example Usage¶

resource "foreman_host" "example" {












  name = "compute01.dc1.company.com"



}

Argument Reference¶

The following arguments are supported:

  • bmc_success - (Optional; schema.TypeBool) Tracks the partial state of BMC operations on host creation. If these operations fail, the host will be created in Foreman and this boolean will remain false. On the next terraform apply will trigger the host update to pick back up with the BMC operations.
  • comment - (Optional; Force New; schema.TypeString) Add additional information about this host.Note: Changes to this attribute will trigger a host rebuild.
  • compute_profile_id - (Optional; Force New; schema.TypeInt)
  • compute_resource_id - (Optional; Force New; schema.TypeInt)
  • domain_id - (Optional; Force New; schema.TypeInt) ID of the domain to assign to the host.
  • enable_bmc - (Optional; schema.TypeBool) Enables PMI/BMC functionality. On create and update calls, having this enabled will force a host to poweroff, set next boot to PXE and power on. Defaults to false.
  • environment_id - (Optional; Force New; schema.TypeInt) ID of the environment to assign to the host.
  • hostgroup_id - (Optional; Force New; schema.TypeInt) ID of the hostgroup to assign to the host.
  • image_id - (Optional; Force New; schema.TypeInt) ID of an image to be used as base for this host when cloning
  • interfaces_attributes - (Optional; Force New; schema.TypeSet of schema.Resource) Host interface information.
  • medium_id - (Optional; Force New; schema.TypeInt) ID of the medium mounted on the host.
  • method - (Optional; Force New; schema.TypeString) Chooses a method with which to provision the HostOptions are "build" and "image"
  • name - (Force New; schema.TypeString) Host fully qualified domain name.
  • operatingsystem_id - (Optional; Force New; schema.TypeInt) ID of the operating system to put on the host.
  • parameters - (Optional; schema.TypeMap) A map of parameters that will be saved as host parameters in the machine config.
  • retry_count - (Optional; schema.TypeInt) Number of times to retry on a failed attempt to register a new host in foreman.

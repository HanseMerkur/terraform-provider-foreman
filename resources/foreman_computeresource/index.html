foreman_computeresource¶

Description¶

Foreman representation of computeresource. ComputeResources serve as an identification string that defines autonomy, authority, or control for a portion of a network.

Interactivity¶

  • Creatable: YES
  • Deletable: YES
  • Mutable: YES

Example Usage¶

resource "foreman_computeresource" "example" {










}

Argument Reference¶

The following arguments are supported:

  • cachingenabled - (Optional; schema.TypeBool) For VMware only
  • datacenter - (Optional; schema.TypeString) For oVirt, VMware Datacenter
  • displaytype - (Optional; schema.TypeString) For Libvirt: "VNC" or "SPICE". For VMWare: "VNC" or "VMRC"
  • hypervisor - (schema.TypeString) The HyperVisor/Cloud Provider for this Compute Resource:supported providers include "Libvirt", "Ovirt", "EC2","Vmware", "Openstack", "Rackspace", "GCE"
  • name - (schema.TypeString) Name of the compute resource
  • password - (Optional; schema.TypeString) Password for oVirt, EC2, VMware, OpenStack. Secret key for EC2
  • server - (Optional; schema.TypeString) For VMware
  • setconsolepassword - (Optional; schema.TypeBool) For Libvirt and VMware only
  • url - (schema.TypeString) URL for Libvirt, oVirt, OpenStack and Rackspace
  • user - (Optional; schema.TypeString) Username for oVirt, EC2, VMware, OpenStack. Access Key for EC2.
